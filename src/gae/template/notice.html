フォーマット

{{text['uc']}}

文字コード: UTF-8

改行コード: CR + LF ( Windows )

日本郵便配布のCSV形式のデータの文字コードだけを変えたものです。

カンマ "," で区切られた各項目についての説明は、 日本郵便の {{ref_a}} と {{ref_b}} をご参照ください。

{{text['t1']}}

文字コード: UTF-8

改行コード: LF ( Unix )

"{{text['uc']}}" をタブ区切り形式に変換したものです。

タブ・コードで区切られた各項目についての説明は、 日本郵便の {{ref_a}} と {{ref_b}} をご参照ください。

{{text['t2']}}

文字コード: UTF-8

改行コード: LF ( Unix )

日本郵便配布の住所データには、長いレコードを複数の行に格納したものがあります。 "{{text['t1']}}" を基に、複数行に格納された論理的な 1レコードが 1行になるよう変換したものです。 事業所データは "{{text['t1']}}" と同じものです。

タブ・コードで区切られた各項目についての説明は、 日本郵便の {{ref_a}} と {{ref_b}} をご参照ください。

{{text['t3']}}

文字コード: UTF-8

改行コード: LF ( Unix )

"{{text['t2']}}" を基に、以下のような加工をしたものです。

  1. 都道府県と市区町村のデータを分離
  2. 住所データの「以下に掲載がない場合」など、住所表記以外の説明の項目を分離
  3. 住所データの丸括弧で括られた詳細な表記「（...）」やビル名・ビルの階層などの項目を分離
  4. 住所のデータと、事業所のデータの項目の構成を統一

各レコードの項目は以下のようになります。事業所データは一部の読みを含みません。

  1. 市区町村コード
  2. 郵便番号
  3. 町名など
  4. 番地・ビル名など
  5. 住所データのビルの場合は階層、事業所データの場合は事業所名
  6. 説明事項
  7. 「3.」の読み
  8. 「4.」の読み
  9. 「5.」の読み
  10. 一町域が二以上の郵便番号で表される場合の表示 : {{ref_a}} 参照
    複数番号の有無 : {{ref_b}} 参照
  11. 小字毎に番地が起番されている町域の表示 : {{ref_a}} 参照
  12. 丁目を有する町域の場合の表示 : {{ref_a}} 参照
  13. 一つの郵便番号で二以上の町域を表す場合の表示 : {{ref_a}} 参照
  14. 更新の表示 : {{ref_a}} 参照
    修正コード : {{ref_b}} 参照
  15. 変更理由 : {{ref_a}} 参照
  16. 個別番号の種別の表示 : {{ref_b}} 参照
  17. 取扱局（漢字） : {{ref_b}} 参照

都道府県データの項目は以下のようになります。

  1. 都道府県コード : 数字2桁
  2. 都道府県名
  3. 都道府県名読み : 事業所データの場合は空欄

市区町村データの項目は以下のようになります。

  1. 市区町村コード : 数字2桁
  2. 市区町村名
  3. 市区町村名読み : 事業所データの場合は空欄

{{text['ja']}}

文字コード: UTF-8

改行コード: LF ( Unix )

自治体別のJSON形式のデータです。JSONについては http://www.json.org/json-ja.html をご参照ください。

フォルダ ( ディレクトリ ) の構成は以下の通りです。 

+-- k                 -- 住所データ
|   |   pref.json     -- 都道府県データ
|   +-- 01            -- 北海道
|   |       city.json -- 北海道の市区町村データ
|   |       101.json  -- 札幌市中央区
|   |       ...
|   |       643.json  -- 中川郡幕別町
|   ...
|   +-- 47            -- 沖縄県
|           city.json -- 沖縄県の市区町村データ
|           201.json  -- 那覇市
|           ...
|           382.json  -- 八重山郡与那国町
+-- j                 -- 事業所データ
    |   pref.json     -- 都道府県データ
    +-- 01            -- 北海道
    ...

{{text['jz']}}

文字コード: UTF-8

改行コード: LF ( Unix )

郵便番号別のJSON形式のデータです。JSONについては http://www.json.org/json-ja.html をご参照ください。

フォルダ ( ディレクトリ ) の構成は以下の通りです。 完全に郵便番号別のファイルにすると膨大なファイル数になり、利用環境によっては問題が起きると思われるため、 郵便番号の上 5桁が共通のものを 1個のファイルに格納しています。 

+-- k                 -- 住所データ
|   |   pref.json     -- 都道府県データ
|   +-- 01            -- 北海道
|   |       city.json -- 北海道の市区町村データ
|   |   ...
|   +-- 47            -- 沖縄県
|   |       city.json -- 沖縄県の市区町村データ
|   +-- 001               -- 郵便番号 上 3桁
|   |       00.json       -- 郵便番号 001-00..
|   |       09.json       -- 郵便番号 001-09..
|   |   ...
|   +-- 999
|           ...
+-- j                 -- 事業所データ
    |   pref.json     -- 都道府県データ
    +-- 01            -- 北海道
    |   ...
    +-- 001               -- 郵便番号 上 3桁
        ...

JSONデータのフォーマットは以下の通りです。 改行とインデントは、ここでの説明のために入れていますので、このとおりではありません。 値の無い項目は、キー値有りの空文字列ではなく、キー値も存在しない形にしています。

住所データ:例: /k/100/00.json

  {

    "p":{
      "c":"13",

    },

      "n":"東京都"
    },
    "c":{
      "c":"13101",
      "n":"千代田区"
    },
    "s":"有楽町"
  }
]
